# ChecklistHub — Quick Guide

## Create a checklist
`Checklists → New`
1. **Details** — name + taxonomic scope + region
2. **Import** — upload CSV or run discovery search
3. **Validate** — review merged species list
4. **Collab** — invite reviewers by email
5. **Create**

## Review species
`Checklist → Workbench`
- Open a species → check evidence, taxonomy, history
- Comment / discuss / vote on conflicts
- Accept or reject (needs ≥1 reviewer)

## Resolve conflicts
`Checklist → Reconciliation`
- Shared / missing species vs. other checklists
- Synonym and taxonomic conflicts flagged here

## Publish
`Checklist → Publish`
1. **Validate** — readiness checklist (all reviewed, no conflicts)
2. **Metadata** — checklist info + contributors
3. **Review** — preview package (DwC-A files)
4. **IPT** — pick publisher org → pick IPT installation → download package → upload to IPT → paste back published URL
5. **Done**

## Watch a checklist
`Checklist → Workbench → Status → Watcher`
- For checklists that stay active after import, where new field records keep coming in
- Weekly/monthly re-fetch from GBIF, iNaturalist (and eBird for Aves), surfacing new candidate species + new observations on existing ones
- Alerts your chosen collaborators by email + in-app notification, with a results view to review and apply
- Nothing applies automatically — a reviewer always confirms first

---
Stuck? Every step shows a readiness/blocker list before letting you proceed — fix what's flagged, then continue.
